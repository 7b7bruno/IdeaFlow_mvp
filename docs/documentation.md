## 6. DATA STRUCTURES DESCRIPTION

This section describes the key database tables, JSON data structures, and TypeScript interfaces used throughout the IdeaFlow application.

### 6.1 Database Schema

IdeaFlow uses SQLite for local data persistence. The database file `ideaflow.db` is stored on the device using expo-sqlite v16.0.8 with WAL (Write-Ahead Logging) mode enabled for optimized concurrent access.

#### Ideas Table

The primary entity in the system is the **Idea** - a recorded voice note with AI-generated transcription and title. Idea information is stored in the `ideas` table:

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | Unique identifier for each idea |
| title | TEXT | NOT NULL | User-editable title (3-6 words), generated by AI or extracted from transcription |
| audioPath | TEXT | NOT NULL | File system path to the recorded audio file (.m4a format) |
| transcription | TEXT | NULL | AI-generated transcription of the audio content |
| createdAt | INTEGER | NOT NULL | Unix timestamp in milliseconds when the idea was created |
| updatedAt | INTEGER | NOT NULL | Unix timestamp in milliseconds when the idea was last modified |

**Indexes:**
- `idx_ideas_createdAt` - Descending index on `createdAt` for efficient chronological queries

**SQL Schema:**
```sql
CREATE TABLE ideas (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  audioPath TEXT NOT NULL,
  transcription TEXT,
  createdAt INTEGER NOT NULL,
  updatedAt INTEGER NOT NULL
);

CREATE INDEX idx_ideas_createdAt ON ideas(createdAt DESC);
```

### 6.3 External API Data Formats

The application communicates with Google Gemini AI API for transcription and title generation. All API requests use JSON format.

#### Transcription API Request

**Endpoint:** `https://generativelanguage.googleapis.com/v1/models/gemini-2.0-flash-001:generateContent`

**Request Body:**
```json
{
  "model": "gemini-2.0-flash-001",
  "contents": [
    {
      "parts": [
        {
          "text": "Transcription cleanup prompt..."
        },
        {
          "inlineData": {
            "data": "<base64-encoded-audio-file>",
            "mimeType": "audio/mp4"
          }
        }
      ]
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| model | string | AI model identifier |
| contents[].parts[0].text | string | System prompt for transcription cleanup |
| contents[].parts[1].inlineData.data | string | Base64-encoded audio file content |
| contents[].parts[1].inlineData.mimeType | string | MIME type of the audio file (audio/mp4, audio/mpeg, etc.) |

**Response Body:**
```json
{
  "candidates": [
    {
      "content": {
        "parts": [
          {
            "text": "Transcribed and cleaned text content..."
          }
        ]
      }
    }
  ]
}
```

#### Title Generation API Request

**Endpoint:** `https://generativelanguage.googleapis.com/v1/models/gemini-2.0-flash-001:generateContent`

**Request Body:**
```json
{
  "model": "gemini-2.0-flash-001",
  "contents": "Generate a 3-6 word title...\n\nTranscription:\n<transcription-text>"
}
```

| Field | Type | Description |
|-------|------|-------------|
| model | string | AI model identifier |
| contents | string | Combined prompt and transcription text |

**Response Body:**
```json
{
  "candidates": [
    {
      "content": {
        "parts": [
          {
            "text": "Generated Title Text"
          }
        ]
      }
    }
  ]
}
```

### 6.4 File System Data

Audio files are stored in the device's document directory using the Expo File System API (v18.3.0).

**Storage Location:**
```
{DOCUMENT_DIRECTORY}/audio/idea-recording-{timestamp}-{random}.m4a
```

**File Naming Convention:**
```
idea-recording-2025-01-24T10-30-45-123Z-abc123.m4a
```

| Component | Description | Example |
|-----------|-------------|---------|
| Prefix | Fixed prefix for all recordings | `idea-recording-` |
| Timestamp | ISO 8601 timestamp with colons replaced | `2025-01-24T10-30-45-123Z` |
| Random Suffix | Random alphanumeric string | `abc123` |
| Extension | Audio file format | `.m4a` |

**Audio File Specifications:**
- Format: M4A (MPEG-4 Audio)
- Codec: AAC
- Quality: HIGH_QUALITY preset from expo-audio
- Max Duration: 5 minutes (300,000ms)
- Min Duration: 3 seconds (3,000ms)
- Max File Size: 20MB (for API upload)

### 6.6 Data Flow Summary

1. **Recording Creation Flow:**
   - User records audio → Audio file saved to file system
   - Create Idea record in database (temporary title)
   - Send audio to Gemini API → Receive transcription
   - Update Idea record with transcription
   - Send transcription to Gemini API → Receive title
   - Update Idea record with generated title

2. **Idea Retrieval Flow:**
   - Query database for all ideas
   - Load file metadata (size, exists)
   - Format display data (preview, date)
   - Render in list or detail view

3. **Idea Deletion Flow:**
   - Delete record from database
   - Delete audio file from file system
   - Update UI to reflect changes

### 6.7 Data Validation Rules

**Idea Entity:**
- Title: Must be non-empty string, 1-60 characters recommended
- Audio Path: Must be valid file system path pointing to existing file
- Transcription: Optional, can be empty or null
- Created/Updated timestamps: Must be valid Unix timestamps in milliseconds

**Audio Recording:**
- Duration: Minimum 3 seconds, maximum 5 minutes
- File size: Maximum 20MB for API upload
- Format: Must be supported by Gemini API (.m4a, .mp4, .wav, .mp3, .flac, .ogg)

**Title Generation:**
- Length: 3-6 words (validated up to 10 words)
- Format: Title case preferred
- Fallback: First 6 words of transcription if AI generation fails
